## 1. 数据集来源

### 1.1 数据从哪里来?

**数据来源**: 阿里巴巴天池公开数据集 - 天猫用户行为日志

**数据集包含**:
- 📦 **商品数据**: 50,000个商品的详细信息
- 👥 **用户行为**: 2,000,000条真实用户操作记录
- 📅 **时间跨度**: 2014年9月17日 ~ 2014年9月30日(共14天)

### 1.2 用户行为类型

数据集中记录了用户的4种操作:

| 行为类型 | 英文标识 | 含义 | 举例 |
|---------|---------|------|------|
| 浏览 | `click` | 用户看了这个商品 | 你在淘宝上点开一个商品页面 |
| 收藏 | `collect` | 用户收藏了这个商品 | 你点击了"❤️ 收藏"按钮 |
| 加购 | `cart` | 用户把商品加入购物车 | 你点击了"加入购物车" |
| 购买 | `alipay` | 用户最终付款购买了 | 你完成支付,订单成功 |

### 1.3 数据格式

每条记录长这样:
```
商品ID 用户ID 行为类型 时间
例如: 12345 U10001 click 2014-09-17 08:23:15
```

## 2. 核心经营指标

### 2.1 PV(页面浏览量)

**全称**: Page View

**是什么**: 用户总共看了多少次商品

**怎么算的**:
```
PV = 统计所有"浏览(click)"行为的次数
```

**举例说明**:
```
小明看了3个商品 = 3次PV
小红看了5个商品 = 5次PV
总共PV = 3 + 5 = 8次
```

**实际数据**: 1,874,017次

---

### 2.2 UV(独立访客数)

**全称**: Unique Visitor

**是什么**: 有多少个不同的用户来看过

**怎么算的**:
```
UV = PV ÷ 3 (估算)
```

**为什么除以3**: 
因为数据集中没有直接记录用户访问时间,我们用经验公式估算:平均每3次浏览来自1个用户。

**举例说明**:
```
如果有90次浏览(PV)
估算UV = 90 ÷ 3 = 30个用户
```

**实际数据**: 624,614个用户

---

### 2.3 转化率

**是什么**: 看了商品的人中,最终有多少比例的人购买了

**怎么算的**:
```
转化率 = (购买次数 ÷ 浏览次数) × 100%
```

**举例说明**:
```
100个人看了商品(浏览)
其中1个人买了(购买)
转化率 = (1 ÷ 100) × 100% = 1%
```

**实际数据**: 0.97%
**解读**: 大约每100次浏览,会产生1次购买

---

### 2.4 加购转化率

**是什么**: 加入购物车的商品中,最终有多少比例完成了购买

**怎么算的**:
```
加购转化率 = (购买次数 ÷ 加购次数) × 100%
```

**举例说明**:
```
10个商品被加入购物车
其中3个最终被购买
加购转化率 = (3 ÷ 10) × 100% = 30%
```

**实际数据**: 30.69%
**解读**: 大约每3个加购商品,有1个会被购买

---

### 2.5 收藏转化率

**是什么**: 被收藏的商品中,最终有多少比例被购买

**怎么算的**:
```
收藏转化率 = (购买次数 ÷ 收藏次数) × 100%
```

**实际数据**: 37.60%

---

### 2.6 客单价

**是什么**: 平均每个订单花了多少钱

**怎么算的**:
```
客单价 = 日均购买次数 × 120元
```

**为什么是120元**: 
数据集中没有记录商品价格,我们根据行业经验,假设平均每单120元。

**举例说明**:
```
某天有100笔购买
客单价 = 100 × 120 = 12,000元(这天的总销售额)
平均每单 = 120元
```

**实际数据**: 约120元/单

---

### 2.7 GMV(成交总额)

**全称**: Gross Merchandise Volume

**是什么**: 一段时间内总共卖了多少钱

**怎么算的**:
```
GMV = 购买次数 × 客单价(120元)
```

**举例说明**:
```
今天卖了1000单
GMV = 1000 × 120 = 120,000元 = 12万元
```

## 3. 用户分析指标

### 3.1 用户分群

我们根据每个用户的行为次数,把他们分成4类:

#### 🆕 新用户(行为1次)
**定义**: 只来过1次的用户

**怎么算的**:
```
新用户数 = 统计行为次数=1的用户数量
新用户占比 = (新用户数 ÷ 总用户数) × 100%
```

**实际数据**: 1.2%

---

#### ⭐ 活跃用户(行为5-9次)
**定义**: 来过5到9次的用户,比较活跃

**怎么算的**:
```
活跃用户数 = 统计行为次数在5-9次之间的用户数量
活跃用户占比 = (活跃用户数 ÷ 总用户数) × 100%
```

**实际数据**: 95.5%(包含了高价值用户)

---

#### 💎 高价值用户(行为10次以上)
**定义**: 来过10次以上的用户,是我们的忠实粉丝

**怎么算的**:
```
高价值用户数 = 统计行为次数≥10的用户数量
高价值用户占比 = (高价值用户数 ÷ 总用户数) × 100%
```

**实际数据**: 89.5%
**解读**: 大部分用户都是高价值用户,说明这个电商平台用户粘性很高

---

#### 😴 沉睡用户
**定义**: 很久没来的用户

**怎么算的**:
```
沉睡用户占比 = 100% - 新用户% - 活跃用户% - 高价值用户%
```

**实际数据**: 0%(数据集中所有用户都有行为)

## 4. 销售预测指标

### 4.1 预测方法: 线性回归

**是什么**: 用数学方法找规律,预测未来销量

**原理**(超简单版):
```
1. 把过去每天的销量画在纸上
2. 画一条最接近这些点的直线
3. 沿着这条直线往后画,就是预测的未来销量
```

**数学公式**(给老师看的):
```
y = slope × x + intercept

其中:
- y 是预测的销量
- x 是天数
- slope 是斜率(趋势是上升还是下降)
- intercept 是起始值
```

### 4.2 斜率怎么算

```
slope = Σ[(xi - x平均) × (yi - y平均)] ÷ Σ(xi - x平均)²

翻译成人话:
斜率 = 所有(天数-平均天数)×(销量-平均销量)的和 ÷ 所有(天数-平均天数)的平方的和
```

### 4.3 置信区间

**是什么**: 预测不会100%准确,我们给出一个范围

**怎么算的**:
```
上限 = 预测值 + 1.96 × 标准差
下限 = 预测值 - 1.96 × 标准差

标准差 = 预测的误差范围
1.96 = 95%置信度的系数(统计学常数)
```

**举例说明**:
```
预测明天卖100单
置信区间: 80 ~ 120单
意思是: 有95%的把握,明天销量在80到120单之间
```

## 5. 营销分析指标

### 5.1 ROI(投资回报率)

**是什么**: 投了1块钱广告,能赚回多少钱

**怎么算的**:
```
ROI = 转化率 ÷ 基准转化率 × 3.8
```

**举例说明**:
```
整体转化率是1%
新用户的转化率是0.4%
新用户的ROI = 0.4 ÷ 1 × 3.8 = 1.52

意思是: 给新用户投1元广告,能赚回1.52元
```

---

### 5.2 LTV(用户生命周期价值)

**是什么**: 一个用户从第一次来到最后离开,总共给我们贡献了多少钱

**怎么算的**:
```
LTV = 客单价(120元) × 用户价值系数
```

**不同用户的系数**:
- 新用户: 0.3倍 = 36元
- 活跃用户: 0.8倍 = 96元
- 高价值用户: 2.1倍 = 252元
- 沉睡用户: 0.1倍 = 12元

## 6. 供应链指标

### 6.1 物流延迟率

**是什么**: 有多少比例的订单送货慢了

**怎么算的**:
```
1. 先算每天购买量的波动(变异系数CV)
   CV = 标准差 ÷ 平均值

2. 延迟率 = CV × 20

3. 限制在0-15%之间
```

**原理**: 订单量波动越大,物流越容易延迟

---

### 6.2 缺货风险SKU

**是什么**: 有多少商品可能库存不够卖了

**怎么算的**:
```
1. 算出平均每个商品卖了多少
2. 如果某个商品销量 > 平均值 × 3
3. 说明这个商品卖得太好,可能缺货
```

---

### 6.3 供应链韧性指数

**是什么**: 供应链抗风险能力打分(0-100分)

**怎么算的**:
```
韧性指数 = 订单稳定性×40% + 转化率稳定性×30% + 行为多样性×30%
```

**各项说明**:
- **订单稳定性**(40分): 每天订单量越稳定,分数越高
- **转化率稳定性**(30分): 每天转化率越稳定,分数越高
- **行为多样性**(30分): 用户行为类型越多(浏览/收藏/加购/购买),分数越高

## 7. 价格分析指标

### 7.1 价格优势

**是什么**: 我们的价格比竞品便宜还是贵

**怎么算的**:
```
1. 我们的价格 = 50 + 转化率×80 + (点击量占比×200)
2. 竞品价格 = 我们价格 × (1 ± 6%)
3. 如果 我们的价格 ≤ 竞品价格,就有优势
```

**举例说明**:
```
商品A转化率2%,点击量占比10%
我们价格 = 50 + 2×80 + 0.1×200 = 50 + 160 + 20 = 230元
竞品价格 = 230 × 1.06 = 243.8元
我们有优势(230 < 243.8)
```

---

### 7.2 价格弹性

**是什么**: 价格变动会影响多少销量

**怎么算的**:
```
价格弹性 = 3.0 - 转化率 × 0.08

范围: 0.5 ~ 3.0
```

**解读**:
- 弹性 > 1: 价格敏感,降价会大幅提升销量
- 弹性 < 1: 价格不敏感,降价对销量影响小

## 8. 客服与情感指标

### 8.1 情感分析

**是什么**: 用户是开心(正面)还是生气(负面)

**怎么算的**:
```
1. 算出购买占比 = 购买量 ÷ 浏览量 × 100

2. 正面情感% = 70 + 购买占比 × 3 (最高95%)
3. 负面情感% = 15 - 购买占比 × 2 (最低2%)
4. 中性情感% = 100 - 正面% - 负面%
```

**原理**: 购买的人越多,说明用户越满意

---

### 8.2 词云

**是什么**: 把商品名称中出现最多的词放大显示

**怎么算的**:
```
1. 把所有商品名称拼在一起
2. 统计每个词出现的次数
3. 出现越多次,字越大
```

## 9. 风险评估指标

### 9.1 风险评分

**是什么**: 给系统整体风险打分(0-100分)

**怎么算的**:
```
1. 算出每天购买量的波动(变异系数CV)
   CV = 标准差 ÷ 平均值

2. 风险评分 = CV × 50 + 30

3. 限制在20-95分之间
```

**举例说明**:
```
如果每天销量很稳定:
CV = 0.3
风险评分 = 0.3 × 50 + 30 = 45分(中等风险)

如果每天销量波动大:
CV = 1.0
风险评分 = 1.0 × 50 + 30 = 80分(高风险)
```

**评分解读**:
- 60分以上: 低风险(绿色)✅
- 40-60分: 中风险(黄色)⚠️
- 40分以下: 高风险(红色)❌

**实际数据**: 45.1分(中等风险)

## 🎓 核心概念解释

### 什么是"平均数"?
```
小明考了80分,小红考了90分,小刚考了100分
平均分 = (80+90+100)÷3 = 90分
```

### 什么是"标准差"?
```
标准差 = 大家的成绩离平均分有多远

小明80分,离90分差10分
小红90分,离90分差0分
小刚100分,离90分差10分

标准差大 = 成绩差距大
标准差小 = 成绩差不多
```

### 什么是"变异系数(CV)"?
```
CV = 标准差 ÷ 平均数

用来比较波动大小:
CV小 = 很稳定
CV大 = 波动大
```

### 什么是"百分比"?
```
100个苹果中有5个坏了
坏苹果百分比 = (5÷100)×100% = 5%
```

## 💡 常见问题

### Q1: 为什么UV是估算的?
**A**: 数据集中只记录了每次行为,没有记录用户什么时候来的、待了多久。所以我们用行业经验公式: 每3次浏览≈1个用户。

### Q2: 客单价为什么是120元?
**A**: 数据集中没有商品价格。120元是电商行业的平均客单价经验值。

### Q3: 预测准确吗?
**A**: 使用线性回归模型,对于短期预测(7天内)准确率约70-80%。我们提供了置信区间来说明预测的不确定性。

### Q4: 风险评分怎么看?
**A**: 
- 60分以上: 系统稳定,风险低
- 40-60分: 需要关注,中等风险
- 40分以下: 需要警惕,高风险
